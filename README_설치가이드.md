# NK 약술논술 AI 케어 — 설치·운영 가이드 (v2)

원장님(비개발자)을 위한 안내서입니다. 순서대로 따라 하시면 됩니다.
준비 시간 약 15~20분, 무료로 시작할 수 있습니다.

---

## 먼저, 설치 없이 둘러보기 (강력 추천)

**가장 간단한 방법:** 아래 데모 주소를 브라우저에서 바로 여세요 (설치 불필요).
- 데모: **https://yaknon.vercel.app/?demo=1**

또는 `index.html` 파일을 더블클릭해서 브라우저로 연 뒤, 주소창 끝에 **`?demo=1`** 을 붙이고 Enter 하세요.
예) `file:///C:/.../index.html?demo=1`

- 인터넷만 연결돼 있으면 **Supabase 설정 없이** 내장 샘플 데이터로 전체 화면을 바로 볼 수 있습니다.
- 화면 우측 상단 **[관리자 / 학생 / 학부모]** 스위치로 세 가지 역할 화면을 모두 미리볼 수 있습니다.
- 데모 모드에서는 입력·저장이 실제로 반영되지 않습니다(둘러보기 전용).

마음에 들면 아래 순서로 실제 설치를 진행하세요.

## 이 프로그램은 무엇인가요?

학생들의 **주간테스트 채점, 과제, 첨삭**을 입력하면 **준비도(%)·취약 진단(히트맵)·합격 가능성·주간 리포트**를 자동으로 만들어 줍니다.
서버가 필요 없고, **HTML 파일 하나**를 브라우저에서 열어 씁니다. 데이터는 무료 클라우드 **Supabase**에 저장됩니다.

**세 가지 역할**
| 역할 | 볼 수 있는 것 |
|------|--------------|
| 관리자(admin) | 전체 학생 관리·채점 입력·분석 (9개 메뉴) |
| 학생(student) | 내 리포트·취약 진단·첨삭 이력·목표 대학 (읽기 전용) |
| 학부모(parent) | 자녀 주간 리포트 (휴대폰 최적화 한 페이지) |

**받으신 파일 3개**
| 파일 | 용도 |
|------|------|
| `index.html` | 프로그램 본체 (더블클릭 실행, 데모 모드 내장) |
| `supabase_setup.sql` | 데이터베이스 초기 설정 스크립트 |
| `README_설치가이드.md` | 이 문서 |

## 설치 순서

1. Supabase 무료 프로젝트 만들기
2. `supabase_setup.sql` 실행 (테이블 생성 + 대학 15개 자동 등록)
3. 관리자 비밀번호 해시 만들기 (`?hashgen=1`)
4. 관리자 계정 등록
5. HTML에 Supabase 주소·키 연결
6. 로그인해서 사용 시작

### 1) Supabase 프로젝트 만들기 (5분)
1. https://supabase.com → **Start your project** → 가입/로그인
2. **New project** → 이름(예: `nk-yaknon`), **Database Password**(메모), Region은 `Northeast Asia (Seoul)` 권장
3. **Create new project** → 1~2분 대기

### 2) 데이터베이스 만들기 (2분)
1. 왼쪽 **SQL Editor** → **New query**
2. `supabase_setup.sql` 파일을 메모장으로 열어 **전체 복사** → 붙여넣기 → **Run**
3. `Success` 메시지가 나오면 완료. (테이블과 함께 약술 논술 15개 대학이 자동 등록됩니다.)

> ⚠️ 이 스크립트 맨 위 "기존 테이블 삭제" 블록은 **재설치용**입니다. 이미 운영 중 데이터가 있다면
> 먼저 백업하거나 해당 DROP 줄들을 주석 처리(줄 앞에 `--`)한 뒤 실행하세요.

### 3) 관리자 비밀번호 해시 만들기 (2분)
1. `index.html` 을 브라우저로 열고 주소 끝에 **`?hashgen=1`** 을 붙여 Enter
2. 로그인용 전화번호·비밀번호 입력 → **[해시 생성]**
3. 아래에 나오는 **완성된 INSERT SQL 을 복사** (다음 단계에서 사용)

### 4) 관리자 계정 등록 (1분)
1. Supabase **SQL Editor → New query** 로 이동
2. 3)에서 복사한 INSERT 붙여넣기 → **Run**
   ```sql
   insert into accounts (phone, password_hash, role)
   values ('01012345678', '(생성된 해시)', 'admin');
   ```
> **빠른 테스트**: `supabase_setup.sql` 맨 아래 주석의 예시(`01012345678` / `nk1234`)를 그대로 실행해도 됩니다. 운영 시엔 꼭 본인 계정으로 새로 만드세요.

### 5) HTML에 Supabase 연결 (3분)
1. Supabase **Project Settings → API** 에서 **Project URL** 과 **anon public** 키 복사
2. `index.html` 을 **메모장**으로 열어 상단에서 아래를 찾습니다:
   ```js
   const SUPABASE_URL = "여기에_붙여넣기";
   const SUPABASE_ANON_KEY = "여기에_붙여넣기";
   ```
3. 큰따옴표는 그대로 두고 값만 교체 후 **저장(Ctrl+S)**

### 6) 로그인
`index.html` 을 더블클릭(또는 F5) → 관리자 전화번호·비밀번호로 로그인. 로그인은 **30일 유지**됩니다.

## 관리자 사용법 (메뉴 11개)

1. **대시보드** — 전 학생을 한 화면에서: 재원 수/반평균/채점 입력률/최근접 D-day, 학생별 최근점수·증감·준비도·밴드·취약 1순위·이번주 입력 여부·**최근 상담일**. 행을 클릭하면 그 학생 취약 진단으로 이동.
2. **학생 관리** — 학생 등록/수정, 목표 대학(최대 3, 지망순위) 연결, **학생·학부모 로그인 계정 생성**.
3. **상담 기록** — 학생별 상담 이력을 남깁니다. 날짜·유형(정기/학부모/진로·지원/기타)·상담 내용·후속 조치 입력, 타임라인으로 조회·수정·삭제. **관리자 전용이며 학생·학부모 화면에는 노출되지 않습니다.**
4. **주간테스트** — 3단계로 진행:
   - ① 회차 관리: 주차·시험일·범위 단원·총점으로 새 회차 생성
   - ② 문항 정의: 문항별 단원·사고과정·배점·출처 입력 (배점 합계가 총점과 맞는지 표시)
   - ③ 채점 그리드: **행=학생, 열=문항**. 셀에 득점 입력(부분점수 가능). **방향키(상하좌우)로 셀 이동**, Enter는 아래로. 셀의 태그 버튼으로 **오답원인** 지정(태그된 셀은 보라 점 표시). 행 합계·문항 정답률 자동 계산 → **[전체 저장]**.
5. **과제 점검** — 숙제 제출 관리는 학원 앱에서 하고, 여기서는 **풀이 채점 결과만** 기록합니다. 날짜·전체 문항 수·맞은 문항 수·걸린 시간·메모 입력 → 정답률·문항당 평균 시간 자동 계산. 상단에 평균 정답률·평균 풀이 시간·최근 4주 추이 표시.
6. **취약 진단** — 6×4 히트맵(최근 3회 가중 50/30/20%, 득점/배점 기반), 우선 보완 순위(실점 기여도), 오답 원인 패턴.
7. **첨삭 관리** — 루브릭 3항목(조건 해석/풀이 과정/최종 답안) 득점·코멘트 입력, 이력 타임라인.
8. **합격 가능성** — 목표 대학별 밴드(도전/적정/안정)·준비도·주간 추이. "시행계획 기준" 배지, 비보장 고지.
9. **주간 리포트** — 1페이지 리포트(KPI: 회차 점수·직전 대비·**과제 정답률**·취약 셀 수, 추이·단원별 정답률·강사 코멘트·다음 주 안내). **[인쇄/PDF]** 로 A4 저장.
10. **대학 정보** — 약술 논술 15개 대학 표. 고사일·모집인원·수능최저·반영비율 등을 인라인 수정.
11. **설정** — 내 비밀번호 변경, 역할 안내, 데이터 관리.

## 학생 사용법
관리자가 만든 전화번호·비밀번호로 로그인. **내 리포트 / 취약 진단 / 첨삭 이력 / 목표 대학** 4개 메뉴, 본인 데이터만 읽기 전용.

## 학부모 사용법
관리자가 만든 학부모 계정으로 로그인. 자녀 **주간 리포트**가 휴대폰 한 페이지로 표시됩니다(주차 선택 가능).

## 준비도(%) 계산

데이터가 없는 항목은 자동 제외하고 남은 비중을 다시 나눠 계산합니다.

| 항목 | 비중 | 설명 |
|------|------|------|
| 주간테스트 EWMA | 30% | 회차 점수(득점/총점)의 지수가중이동평균(최근 가중) |
| 기출유형 커버리지 | 25% | 6개 단원 중 채점 기록이 있는 단원 비율 |
| 마스터리(득점) | 20% | 전체 문항의 (Σ득점 / Σ배점) |
| 첨삭 점수 | 15% | 루브릭 총득점/총만점의 평균 |
| 과제 수행 | 10% | 각 과제 점검 기록의 정답률(맞은 문항/전체 문항)의 평균 |

## 문제 해결

- **로그인 버튼이 회색** → 5)의 주소·키 입력/저장을 확인하고 새로고침.
- **"인터넷 연결이 필요합니다"** → 외부 라이브러리·DB 접속에 인터넷이 필요합니다.
- **로그인 실패** → 전화번호는 숫자만. 계정 생성 시 값과 정확히 일치해야 합니다.
- **채점 저장 시 빨간 셀** → 배점 범위를 벗어난 점수입니다. 해당 셀을 수정하세요.
- **비밀번호 분실** → 관리자: `?hashgen=1` 로 새 해시 생성 후 `update accounts set password_hash='새해시' where phone='전화번호';`. 학생/학부모: [학생 관리]에서 재설정.

## ⚠️ 보안 안내 (꼭 읽어주세요)

- 이 프로그램은 **단일 학원 내부용**입니다.
- HTML의 **anon(public) 키**와 데이터 정책(RLS)은 화면 역할 구분만 담당하며, DB는 이 키를 아는 사람이면 접근 가능한 **전체 허용(permissive)** 설정입니다. **HTML 파일과 키를 외부에 공개하지 마세요.**
- 외부 배포·대량 개인정보 보관 등에는 Supabase Auth + 엄격한 RLS로 재설계가 필요합니다.
- 학생 개인정보는 최소화하고, 개인정보 동의일을 기록하세요.
- 대학 정보(반영비율·수능최저·고사일·모집인원)는 **시행계획 기준 예시**이며, 각 대학 최종 모집요강으로 반드시 확인·갱신하세요.

문의나 오류는 화면 캡처와 함께 담당자에게 전달해 주세요.
